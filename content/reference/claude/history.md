# Claude Codeの進化と「今が旬」な理由

Claude Codeは2025年2月のプレビューローンチ以来、驚異的なスピードで進化を続けています。2025年中だけで**176回のアップデート**がリリースされました。

このページでは、Claude Codeがどのように賢くなってきたか、そして今が使い始めるベストタイミングである理由を解説します。

---

## モデルの進化（大きなジャンプ）

Claude Codeの「頭脳」となるモデルは、以下のように進化してきました。

| 時期 | モデル | 主な変化 |
|------|--------|----------|
| 2025年2月 | Sonnet 3.7 | プレビューローンチ。AIペアプログラミングの始まり |
| 2025年5月 | Claude 4 | ツール使用が大幅に改善。モデルがハーネスの力を真に活かせるレベルに |
| 2025年9月 | Sonnet 4.5 + Claude Code 2.0 | システムプロンプトが短くなり、モデル自体が賢くなったため指示が不要に |
| 2025年12月 | **Opus 4.5** | 大きなターニングポイント。複雑な設計判断や俯瞰的な意思決定が飛躍的に向上 |
| 2026年1月末〜 | Opus 4.6（現行） | コーディング能力がさらに強化。1Mトークンコンテキストウィンドウ（ベータ） |

### Opus 4.5が変えたもの

2025年12月のOpus 4.5は、Claude Codeにとって特に大きなターニングポイントでした。

それまでClaude CodeはSonnet系がデフォルトでしたが、Opus系は明らかに以下の点で優れています：

- **大きなリファクタリング**の設計
- **複数ファイルにまたがる変更**の整合性
- **曖昧な指示から意図を汲み取る力**
- **「なんとなく良い感じにして」が通じる度合い**

---

## モデル以外の改善（じわじわ賢くなる部分）

「同じモデルなのにどんどん良くなっている」と感じることがあるかもしれません。それはモデル以外にも複数のレイヤーで継続的に改善されているからです。

### システムプロンプトの継続的チューニング

110以上のプロンプト文字列が常に調整されており、リリースごとに微調整されています。

例えば、専用ツール（Read, Edit, Glob, Grep）をbashの代わりに使うようモデルを誘導する改善などが行われています。

### コンテキスト管理の改善

auto-compact（自動圧縮）のしきい値が下げられ、より多くの「作業用メモリ」を確保するようになりました。

以前は90%以上までコンテキストを使い切っていましたが、現在は64-75%あたりで早めにcompactするようになり、推論品質が維持されるようになりました。

### ツールの追加・改善

以下のような機能が順次追加され、エージェントとしての振る舞いが洗練されてきました：

- **Plan mode**: 実装前に計画を立てて承認を得る
- **Subagents**: 複数のエージェントが並列で作業
- **/context コマンド**: コンテキストの管理
- **Skills**: 繰り返しタスクの自動化
- **CLAUDE.md**: プロジェクト固有の設定

### ホットフィックスのサイクル

一時的にプロンプトで対処 → 次のモデルで根本解決、というサイクルが回っています。

例えば、Sonnet 3.7時代には「gotcha（ハマりポイント）対策」として、よくある失敗パターンを回避するための指示がシステムプロンプトに多数含まれていました。「bashでファイルを読まないで専用ツールを使って」「勝手にコミットしないで」といった注意書きです。

Claude 4.0ではこれらの指示が全て削除され、モデル自体の学習（RLHF: 人間のフィードバックによる強化学習）で対処されるようになりました。つまり、モデルが「何をすべきでないか」を自然に理解するようになったのです。

---

## なぜ「今が旬」なのか

### 1. モデルの成熟

2025年2月のSonnet 3.7から2025年12月のOpus 4.5まで、実質的に数世代分のモデルジャンプが起きています。当初の「面白いけど実用には...」というレベルから、「本当に仕事で使える」レベルに到達しました。

### 2. ハーネスの洗練

176回のアップデートを経て、システムプロンプト・コンテキスト管理・ツール設計が磨き上げられています。

### 3. 長いセッションでも品質が落ちにくい

Opus 4.6の1Mトークンコンテキストウィンドウ（ベータ）により、長時間の開発セッションでも品質が維持されます。

### 4. プログラミング経験がなくても使える

モデルが「曖昧な指示から意図を汲み取る力」を獲得したことで、プログラミング経験がない人でも自然な言葉で指示を出せるようになりました。

---

## まとめ

Claude Codeは、モデルの切り替え以外にも、システムプロンプト・コンテキスト管理・ツール設計の改善が重なって「どんどん賢くなる」体験を生んでいます。

特に2025年12月のOpus 4.5への移行は、モデルチェンジ以上に体感の差が大きかったという声も多く、**今がClaude Codeを使い始めるベストタイミング**と言えます。

このワークショップで、その「旬」のClaude Codeを体験してください。
