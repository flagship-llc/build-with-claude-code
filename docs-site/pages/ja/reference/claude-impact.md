# Claudeが変える世界

AIコーディングツールの登場により、ソフトウェア開発の世界は大きく変わりつつあります。

---

## 「SaaS is Dead」の時代

### 従来のSaaSモデル

これまでソフトウェアを使うには、既存のSaaS（Software as a Service）を契約するのが一般的でした。

- 月額料金を払って既存のツールを使う
- 機能は提供者が決める
- カスタマイズには限界がある
- 「ちょっと違うんだよな...」を我慢する

### AIが変えたこと

Claude Code のようなAIコーディングツールの登場により、状況が変わりました。

**「買う」から「作る」へ**

- 自分のニーズに完全に合ったツールを作れる
- プログラミング経験がなくても開発できる
- 既存SaaSの月額料金より安く済むことも
- 「こんなツールがあったらいいな」を実現できる

### Cowork の例

[Cowork](https://www.cowork.so/) は、Claude を使って自分専用のツールを作る新しいアプローチを提唱しています。

- 既存のSaaSを使う代わりに、AIと一緒にツールを作る
- 自分のワークフローに完全にフィットしたものが手に入る
- 一度作れば、月額料金なしで使い続けられる

---

## 開発の民主化

### 「作る」が誰でもできる時代に

かつてソフトウェア開発は、専門的なスキルを持つエンジニアだけのものでした。

**Before（AIコーディング以前）**:
- プログラミング言語を学ぶ必要がある
- 数年の学習と経験が必要
- ツールを作りたくても外注するしかない

**After（AIコーディング以降）**:
- 自然な言葉で「こういうものを作って」と伝えれば動く
- プログラミング経験がなくても開発できる
- 自分でツールを作る選択肢が生まれた

### 誰が開発者になれるのか

- **ビジネスパーソン**: 自分の業務を効率化するツールを作る
- **研究者**: データ分析や実験自動化のスクリプトを作る
- **クリエイター**: 制作ワークフローを支援するツールを作る
- **すべての人**: 日常の「面倒」を解決するツールを作る

## 「自分専用ツール」という選択肢

### 既存SaaSの限界

| 既存SaaS | 自分で作るツール |
|----------|-----------------|
| 機能は提供者が決める | 自分のニーズに完全にフィット |
| 月額料金が発生し続ける | 一度作れば追加コストなし |
| カスタマイズに限界 | 自由にカスタマイズ可能 |
| 不要な機能も含まれる | 必要な機能だけ |

### 「作る」が現実的になった理由

1. **AIの推論能力の向上**: Claude Opus 4.5/4.6 は複雑な設計判断ができる
2. **開発環境の整備**: VS Code + Claude Code で誰でも始められる
3. **コストの低下**: 月額 $20 で本格的な開発ができる

## このワークショップの意味

### 新しいスキルを身につける

AIとのペアプログラミングは、これからの時代の基礎スキルになりつつあります。

- **問題を言語化する力**: 何を作りたいかを明確に伝える
- **AIと協働する力**: 対話しながら形にしていく
- **技術を活用する力**: ツールを使いこなす

### 選択肢が広がる

ワークショップを通じて身につけたスキルは、仕事でも趣味でも活かせます。

- 「既存のツールで我慢する」から「自分で作る」へ
- 「エンジニアに頼む」から「まず自分で試す」へ
- 「できない」から「やってみる」へ
